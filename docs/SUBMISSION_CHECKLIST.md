# Qwen Hackathon pre-submit gate

## Judge route

1. Open `/` and read the source tiles: Slack, Drive, GitHub, and verified web are server-derived status, not optimistic client labels.
2. Click **Run incident-to-release check**.
3. Show the ordered server trace: Slack incident → Drive runbook → GitHub change → Qwen Reality Memory → deterministic SAG → human owner.
4. Expand Audit proof for source hashes, timestamps, freshness, memory lineage, Qwen status, and MCP `DecisionBrief`.
5. Open `/play/workflow` to show the browser-private MCP lab, then use Money Safety and Rollout Safety as concise reuse proof.

## Must be true before submission

- [ ] The public repository, MIT license, written summary, architecture, live URL, and 1–3 minute video are accessible.
- [ ] `/api/demo/readiness` shows the deployed build SHA, Qwen health, scenario version, and canonical counts.
- [ ] The root route is reachable over HTTPS and works from a fresh browser session.
- [ ] `GET /api/integration-catalog` labels each source with its actual runtime status.
- [ ] An authenticated MCP client can initialize, list tools, run `inspect_memory`, `query_evidence`, and `evaluate_workflow`; an invalid or cross-org key fails.
- [ ] Source status is only called `connected` after its server configuration is complete. If Slack/Drive are not configured, the UI says `setup_required`.
- [ ] The public replay uses the sandbox fixture only; it never modifies canonical memory or counts.
- [ ] Slack HMAC, GitHub signing, Drive folder scope, web SSRF rejection, memory supersession, outcome gating, and no-external-action tests pass.
- [ ] A redacted Alibaba Workbench Overview capture and deployed health/readiness capture are linked from deployment materials.

## Claims to keep precise

- Qwen compiles evidence where the key is configured. The UI exposes `unavailable` or failure rather than inventing a compilation.
- Slack, Drive, GitHub, and verified web are source-specific, least-privilege adapters. This is not a generic connector marketplace or browser credential flow.
- MCP returns governed recommendations and provenance. It cannot perform company actions.
- OAuth 2.1, per-company secret-vault onboarding, general RBAC, and a full connector marketplace are post-hackathon work.
- Hardware attestation is claimed only on a host that actually verifies it; standard hosts use the explicit RSA audit fallback.
